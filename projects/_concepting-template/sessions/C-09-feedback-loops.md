# Session C-09 — Feedback Loops
**Type**: Cross-Agent Retrospective (mandatory)
**Trigger**: C-08 Design QA flags + any cross-pollination conflicts identified during C-01–C-07
**Phase**: 2 — Concepting

---

## Why This Session Exists

Design sessions run sequentially. The Brand Guardian doesn't talk to the Rapid Prototyper. The Narrative Designer doesn't know what the UX Architect decided about transition timing. Conflicts accumulate silently until C-08 catches them. This session routes those conflicts back to the responsible agents for resolution before Phase 3 begins.

**This session is mandatory. Do not skip it.**

---

## Loop Structure

For each conflict or finding that requires a revised answer:

```
Loop [N]: [TITLE]
──────────────────────────────────────────────
Trigger:
  [What finding, decision, or conflict opened this loop]

Routing:
  From: [Agent/session that raised the issue]
  To:   [Agent/session that must respond]

Question:
  [The specific question that needs a revised answer]

Revised Answer:
  [The responding agent's updated position — must be actionable and specific]

Impact:
  [What changes in which prior session — name the file and section]
──────────────────────────────────────────────
```

**Minimum 3 loops.** If fewer than 3 genuine conflicts exist, the sessions were either too thin or conflicts are being suppressed. In either case, escalate to the project owner before closing the phase.

---

## Phase Closure Checklist

Before closing Concepting and passing to Phase 3 (Specification):

- [ ] All C-08 Design QA unresolved items have been looped and resolved
- [ ] All Phase 1 pre-conditions are confirmed addressed in the design
- [ ] Engineer Handoff Checklist from C-07 is fully checked
- [ ] No CRITICAL new risks introduced without a mitigation plan
- [ ] Editorial Filter confirmed applied to all copy in C-04
- [ ] AI image prompts in C-06 are ready to run (no placeholders)

When all boxes are checked: **Concepting phase is complete. Proceed to Phase 3: Specification.**
