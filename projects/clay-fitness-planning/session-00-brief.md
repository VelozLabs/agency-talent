# Session 00 — Idea Brief
**Project**: Clay — Personal Fitness App
**Date**: 2026-03-27
**Status**: Input (human-authored)

---

## Idea in One Sentence

Clay is a personal fitness app built around a single metaphor — one continuous piece of moldable clay — where the interface stretches, compresses, and reshapes around the user's real life instead of forcing the user to reshape around it.

## Founder Context

- Primary user persona: Harold — follows ladder programming, experienced lifter, busy schedule
- Core emotional anchor: personal "why" (e.g., "get back to playing pickup ball with the kids")
- Philosophy: fitness intelligence should whisper, not shout; adapt, not dictate

## The Differentiating Bet

No tabs, no grids, no separate screens. One continuous canvas. The biggest element on screen changes based on what matters *today*. Routine editing lives inside the same clay — not locked in an "edit program" corner. The app absorbs friction; it doesn't create it.

## The Metaphor as UX Contract

- Dashboard **stretches** forward into a workout
- Flows **back** when you finish
- **Pinch-to-compress**: trims accessory work, prioritizes compounds, adjusts volume without breaking weekly progression
- **Long-press to soften**: lift stretches into 2–3 variation options with history attached
- **Voice-note on set**: attach a note directly to a logged set — no menus

## Voice & Tone Constraints

- Honest, adult, direct — never hype, never shame
- No streaks, no "crush it," no passive-aggressive missed-workout nudges
- No confetti or gamification
- Data whispers, not cheerleading

## Technical Ambitions

- Apple HealthKit (priority): read + write back
- Oura / WHOOP: optional enrichment layer
- On-device first, seamless watch sync, one-tap full export
- Programming intelligence layer: classifies any methodology (linear, ladder, undulating, block, etc.)
- Ladder-specific intelligence: detects accommodation stalls, underloading signals
