# Session 00 — Idea Review & Validation Brief
**Project**: Idea Review & Validation
**Date**: 2026-03-31
**Status**: Awaiting Ideas (3 ideas to be submitted)

---

## Purpose

Evaluate three ideas through a structured validation process covering market viability, technical feasibility, user demand, competitive positioning, business model strength, and execution risk. Each idea receives an independent assessment before a comparative ranking.

## Validation Team (9 Agents)

| # | Agent | Role in Validation |
|---|---|---|
| 1 | **Product Trend Researcher** | Market intelligence — scans trends, competitive landscape, and whitespace for each idea |
| 2 | **Product Feedback Synthesizer** | User signal analysis — synthesizes existing user pain points and demand signals relevant to each idea |
| 3 | **Design UX Researcher** | User validation — evaluates target user needs, usability assumptions, and product-market fit indicators |
| 4 | **Engineering Software Architect** | Technical feasibility — assesses architecture complexity, integration risks, and build cost |
| 5 | **Engineering Rapid Prototyper** | MVP scoping — estimates minimum viable scope, prototype timeline, and quick-win opportunities |
| 6 | **Marketing Growth Hacker** | Growth potential — evaluates acquisition channels, viral mechanics, and scalability of each idea |
| 7 | **Sales Deal Strategist** | Business viability — scores monetization model, competitive positioning, and willingness-to-pay |
| 8 | **Project Management Experiment Tracker** | Validation design — defines the experiments and metrics needed to de-risk each idea |
| 9 | **Testing Reality Checker** | Final verdict — stress-tests claims, flags unsupported assumptions, delivers go/no-go recommendation |

## Validation Framework

Each idea will be evaluated across six dimensions (scored 1–5):

| Dimension | Lead Agent(s) |
|---|---|
| **Market Opportunity** | Trend Researcher, Growth Hacker |
| **User Demand & Fit** | Feedback Synthesizer, UX Researcher |
| **Technical Feasibility** | Software Architect, Rapid Prototyper |
| **Business Model Strength** | Deal Strategist, Growth Hacker |
| **Execution Risk** | Experiment Tracker, Reality Checker |
| **Differentiation & Defensibility** | Trend Researcher, Deal Strategist |

## Scoring Output (Session 08)

Each idea receives:
- A 1–5 score per dimension (6 dimensions)
- A weighted composite score
- A go / no-go / pivot recommendation
- Top 3 risks and suggested de-risking experiments
- A comparative ranking across all 3 ideas
